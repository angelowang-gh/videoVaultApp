# 浏览器文件夹选择功能说明

## 📋 功能概述

VideoVault 提供两种添加视频文件夹的方式：

1. **浏览器文件夹预览**（新增）- 通过浏览器原生界面选择文件夹进行预览
2. **手动输入路径**（推荐）- 直接输入文件系统路径用于永久添加

## ⚠️ 重要说明

### 浏览器安全限制

由于浏览器的安全沙箱机制，**浏览器无法直接将选择的文件夹添加到系统的扫描路径列表中**。这是因为：

1. **无法获取完整路径**：浏览器不会暴露文件的绝对路径（如 `C:\Videos`）
2. **无法持久化访问**：File API 返回的对象在页面刷新后失效
3. **后端无法访问**：服务器无法直接读取浏览器选择的文件

### 当前实现

"Preview Folder (Browser)" 按钮的功能是：
- ✅ 扫描并统计文件夹中的视频文件数量
- ✅ 验证文件夹是否包含视频
- ✅ 在控制台输出文件列表（供调试）
- ❌ **不能**将文件夹添加到左侧目录
- ❌ **不能**持久化保存

**要永久添加文件夹，请使用"Manual Path Input"输入完整路径。**

## 🎯 浏览器文件夹选择器

### 使用方法

1. 打开设置（点击右上角齿轮图标）
2. 在 "Video Folders" 部分，点击 **"Select Folder (Browser)"** 按钮
3. 在弹出的文件选择对话框中：
   - 选择一个包含视频的文件夹
   - 浏览器会递归扫描该文件夹及其所有子文件夹
4. 系统会自动识别并统计视频文件数量

### 技术实现

使用 HTML5 File API 和 `webkitdirectory` 属性：

```html
<input 
  type="file" 
  webkitdirectory 
  directory 
  multiple 
  accept="video/*"
/>
```

### ⚠️ 重要限制

#### 1. 无法持久化访问

**问题**：由于浏览器安全沙箱限制，File API 返回的 `File` 对象无法在页面刷新后继续使用。

**影响**：
- ❌ 每次刷新页面后需要重新选择文件夹
- ❌ 无法在后台持续监控文件夹变化
- ❌ 关闭浏览器后失去访问权限

**解决方案**：如需持久化访问，请使用"手动输入路径"方式。

#### 2. 无法获取完整路径

**问题**：浏览器出于安全考虑，不会暴露文件的完整绝对路径。

**影响**：
- 只能获取相对路径（`webkitRelativePath`）
- 后端无法直接访问这些文件
- 需要在前端处理文件（如提交或生成预览）

#### 3. 大文件夹性能问题

**问题**：选择包含大量文件的文件夹可能导致：
- 浏览器响应变慢
- 内存占用增加
- 处理时间较长

**建议**：
- 避免选择包含数千个文件的根目录
- 尽量选择不超过 500 个视频文件的文件夹
- 耐心等待处理完成

### ✅ 适用场景

浏览器文件夹选择适合以下场景：

1. **临时浏览**：快速查看某个文件夹的视频
2. **隐私保护**：不想暴露完整文件系统路径
3. **跨平台兼容**：在不同操作系统上使用
4. **沙箱环境**：受限的浏览器环境

### ❌ 不适用场景

以下场景建议使用"手动输入路径"：

1. **长期使用**：需要持久化保存文件夹配置
2. **自动扫描**：希望应用自动检测新视频
3. **大库管理**：管理大量视频文件
4. **后端处理**：需要服务器端处理视频（缩略图、元数据等）

## 💡 使用建议

### 推荐工作流程

**方案 A：混合使用**
```
1. 使用"浏览器选择"快速预览文件夹内容
2. 确认无误后，使用"手动输入路径"添加为永久文件夹
3. 享受完整的后端功能（缩略图、元数据等）
```

**方案 B：纯浏览器模式**（适合隐私敏感用户）
```
1. 每次使用时选择文件夹
2. 在前端完成所有操作
3. 关闭页面后不留痕迹
```

### 最佳实践

1. **选择合适的文件夹**
   - 避免选择系统根目录（如 C:\, D:\）
   - 选择专门的视频文件夹
   - 确保文件夹结构清晰

2. **注意文件大小**
   - 单个视频文件建议不超过 2GB
   - 总文件数建议不超过 500 个
   - 超大文件夹请分批处理

3. **浏览器兼容性**
   - ✅ Chrome/Edge（最佳支持）
   - ✅ Firefox（支持）
   - ⚠️ Safari（部分支持）
   - ❌ IE（不支持）
